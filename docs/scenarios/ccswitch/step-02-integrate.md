# Step 02 - 接入

目标：把 `ccswitch` 的调用链路与 sub2api 的输入/输出关系写成“照做即可复现”的步骤。

## 1) 画清楚数据流（建议先写一句话）

建议你用一句话固定下来，例如：

> 上游订阅 → sub2api 转换 → ccswitch 拉取/触发 → 产物提供给客户端

## 2) 接入步骤（按你实际情况填写）

把下面小节当作清单，你只需要把“你是怎么做的”写清楚即可：

### 2.1 sub2api 的输入

- 输入是什么（订阅链接/本地文件/其他）
- 是否做了聚合、过滤、重写（写出关键规则/参数）

### 2.2 sub2api 的输出

- 输出是什么（API URL/文件路径）
- 输出格式（写清楚给哪个客户端/工具用）

### 2.3 ccswitch 如何调用/集成

- ccswitch 配置项位置（截图最好）
- 如何触发更新（定时/手动/Hook）
- 失败时哪里看日志

!!! tip "写法建议"
    每个小节尽量落到“可验证结果”：执行完这一步，你能看到什么现象（界面/日志/HTTP 返回/文件生成）。
