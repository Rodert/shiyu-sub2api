# Step 01 - 准备

目标：把环境信息与“成功判定标准”先写清楚，后面每一步才不会变成玄学。

## 1) 写清楚你的环境

把下面的占位符替换成你的真实信息：

- `ccswitch`：版本 / 部署方式 / 入口（URL 或命令）
- `sub2api`：部署方式 / 入口（URL 或命令）
- 你的订阅来源：一个或多个订阅链接（注意脱敏）
- 你的输出目标：希望产出什么格式、给谁用

## 2) 明确“成功标准”

建议用 3 条可验证标准（示例）：

- 能稳定产出一个可访问的转换结果（HTTP 200 / 文件存在）
- 产物可以被目标客户端导入并正常使用
- 遇到失败能定位：是上游订阅问题 / sub2api 转换问题 / ccswitch 调用问题

## 3) 准备截图/录屏（可选但强烈推荐）

把关键界面截图放到 `docs/scenarios/ccswitch/assets/`，后续每步都能引用，读者学习成本会大幅下降。
