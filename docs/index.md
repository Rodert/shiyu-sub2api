# shiyu-sub2api

把 **sub2api** 的不同用法按「场景」拆开写清楚：你点进一个场景，就能按 **Step 01 → Step N** 跑通，并且遇到问题能在同一栏里快速排查。

## 你会在这里看到什么

- **基础用法**：最小可用、常见配置、输出与组织方式
- **场景整合**：例如与 `ccswitch`、以及其他工具的联动（每个场景都有独立的步骤页）
- **部署与运维**：发布、更新、排障、可观测性

## 我怎么维护这套文档

- 新增一个场景：在 `docs/scenarios/<tool>/` 里新增一个目录与步骤页面
- 把侧边栏加上对应条目：改 `mkdocs.yml` 的 `nav`

## 约定：资源（图片/视频）

- 图片统一放在 `docs/assets/`（或放在场景目录下的 `assets/`）
- 视频建议优先用外链托管；如果一定要内嵌，参考「ccswitch 场景」里的示例写法
